# Shadow5hell

![shadow5hell](https://github.com/user-attachments/assets/61942a26-1423-4832-88ca-a89ac6e3ac51)

Shadow5hell is a lightweight Bash-based terminal monitoring tool that records every command executed in a Linux terminal session along with its output. The collected logs are automatically stored and compressed into a password-protected archive when the terminal session ends.

This project demonstrates how Bash shell features such as command history, traps, and output redirection can be used to create a simple command auditing system.

# 1. Introduction

Linux provides powerful tools for managing systems through the command line. However, there are situations where it is useful to record terminal activity for auditing, learning, debugging, or security analysis.

Shadow Terminal is a simple Bash script that logs:

* commands executed in the terminal
* command outputs
* timestamp of execution
* username
* working directory

At the end of the session, the log file is automatically compressed into a password-protected ZIP archive.

This makes it possible to safely store terminal activity logs without leaving readable files on the system.

The project is intentionally designed to be simple so that beginners can study how shell scripting works.

# 4. Usage Guide

When a new terminal session starts, you will see the following prompt:

```text
 Enable Shadow Terminal? (y/n):

```

If you choose **y**, the logging system will start.

You will then be prompted to enter a password:

```text
Enter Shadow_Log Password:

```

After restarting the terminal, the Shadow Terminal script will load automatically.
This password will be used to protect the compressed log file.
After this, all commands executed in the terminal will be logged until the terminal session ends.

#### Example:

```shell
Enable Shadow Terminal? (y/n): y
Enter Shadow_Log Password: 
Linux@ubuntu:~# pwd
/home/user1
Linux@ubuntu:~# 
```


# 5. Example Output

Example terminal commands:

```
ls
pwd
whoami
```

Example log entry:

```bash
-----------------------------------
TIME    : Thu Mar 5 01:39:12 PM
USER    : user1
DIR     : /home/user1
COMMAND : ls
OUTPUT  :
file1.txt
file2.txt
```

Each command is recorded with its output and execution details.


# 6. Log File Location

Logs are stored in the following directory:

```bash
~/.shadow_logs/
```

Example generated file:

```bash
shadow_2026-03-05_13-30-20.log
```

When the terminal session ends, the log file is automatically compressed into:

```bash
shadow_2026-03-05_13-30-20.log.zip
```

The original log file is deleted, leaving only the protected archive.


# 7. Features

Shadow Terminal includes the following capabilities:

#### Command logging
  Every command typed in the terminal is recorded.

#### Output capture
  The output generated by commands is saved along with the command itself.

#### Timestamp tracking
  Each command entry includes the exact time of execution.

#### User identification
  The username executing the command is recorded.

#### Directory tracking
  The current working directory is logged for each command.

#### Automatic log compression
  When the terminal session ends, the log file is compressed automatically.

#### Password protected archive
  Logs are stored in a ZIP archive protected by a password chosen by the user.

Duplicate command filtering
Repeated commands are not logged multiple times unnecessarily.
Lightweight implementation
The tool is written entirely in Bash and requires minimal dependencies.

# 8. How Shadow Terminal Works

Shadow Terminal integrates with the Bash shell and uses built-in shell mechanisms to observe commands.

The main components involved are:

#### Command History
The script reads the latest command from the Bash history.

#### Shell Traps
The DEBUG trap allows the script to run before each command execution.

#### Output Redirection
The script redirects terminal output using the tee command to capture command results.

#### Session Exit Detection
An EXIT trap is used to trigger compression of the log file when the terminal closes.

These mechanisms together allow the script to monitor commands in real time.

# 9. Security Considerations

Although the logs are password protected, users should be aware of the following:

* ZIP password protection is not as strong as modern encryption standards.
* Sensitive information typed in the terminal (such as passwords in commands) will also be logged.
* Logs are stored locally, so access control to the system is still important.
* This tool should be used for learning, monitoring, or auditing purposes rather than high-security environments.

# 10. Limitations

Shadow Terminal currently has some limitations:

* Only works in Bash shell environments.
* Commands executed before the script loads are not recorded.
* Some interactive programs may not produce clean log output.
* ZIP password protection is basic encryption.
* These limitations are acceptable for a lightweight monitoring script but could be improved in future versions.

# 11. Use Cases

Shadow Terminal can be useful in several scenarios:

* Learning how Bash shell traps work
* Studying Linux command usage patterns
* Recording terminal sessions for debugging
* Basic command auditing on personal systems
* Educational demonstrations of shell scripting techniques

# 12. How the Script Works Internally

The script follows this workflow:

1. Ask the user whether to enable logging.

2. Create a hidden log directory inside the user's home directory.

3. Generate a timestamped log file.

4. Capture commands using the DEBUG trap.

5. Extract the command from Bash history.

6. Append command information to the log file.

7. Capture command output using tee.

8. When the terminal exits, compress the log into a password protected ZIP archive.

9. Remove the original log file.

This simple mechanism allows continuous monitoring without modifying the Bash source code.
